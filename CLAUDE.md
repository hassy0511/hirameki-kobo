# このプロジェクトでのClaudeの役割

## 役割分担

このプロジェクトでは、Claudeは原則として方針設計とレビューを担当し、実装はCodexが担当する。

ユーザーから明示的に実装を依頼されない限り、Claudeはコードを書き換えず、次の形で提案する。

1. 現状の課題
2. 体験としての狙い
3. 変更する画面・遊び・文章
4. 学習上の理由
5. 実装時の完了条件

## 現在の正本

- 世界観と体験：`WORLD_BIBLE_V11.md`
- ゲーム仕様：`GAME_SPEC.md`
- 足し算の学習順序：`docs/curriculum_math_g1_addition.md`
- 引き算の学習順序：`docs/curriculum_math_g1_subtraction.md`
- ステージ一覧：`docs/stage_overview.md`

この5ファイルとユーザーの最新指示が食い違う場合は、ユーザーの最新指示を優先する。

## してはいけないこと

- 旧版の海賊、船長、宝探し、海図、戦闘、ダジャレ中心の会話を復活させない
- 旧版のSVG、会話、画面構成を流用しない
- 学習目標が不明なまま、色や飾りだけを増やさない
- 子どもの不正解をキャラクターが笑わない
- 既存の1画面を変更するときに、ホーム・設計図・結果・図鑑とのつながりを無視しない

## 設計の判断基準

- 子どもが次に何をすればよいか、3秒で分かるか
- 正解した瞬間に世界が変わったと感じるか
- 同じ操作が続きすぎないか
- 絵が学習の助けになっているか
- 文章は小学1年生が一度で読めるか
- 旧版を知らなくても物語と目的が伝わるか

## 返答形式

プロジェクトに関する返答の最後に、ユーザーが次に判断・確認することを `依頼事項` として短く書く。確認不要なら `依頼事項：なし` とする。
