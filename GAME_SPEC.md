# ゲーム仕様 — ひらめき工房 v17

## 製品概要

- タイトル：ひらめき工房
- 対象：小学1・2年生
- 学習範囲：学年ごとに算数6ライン・66ステージ、合計132ステージ
- 形式：ブラウザで動くPWA
- 主対象端末：iPad。スマートフォンとPCにも対応
- 公開先：<https://hassy0511.github.io/hirameki-kobo/>
- 外部ライブラリ、外部画像、学習中の外部通信は使用しない

このv17仕様は、小学1年生と小学2年生をコース選択から遊べる実行版と、小学1年生へ全体ストーリー・出題契約を統合した状態を表す。小学3年生は `docs/curriculum_math_g3_overview.md` と `grade3-curriculum.js` でカリキュラム設計済みだが、問題ビルダーと7ラインUIの実装までは実行時に接続しない。

v16では、生成した工房キービジュアル、キャラクター場面、長さの比較方法アトラス、部品アトラスを共通デザインへ統合した。絵文字や抽象記号を意味の唯一の手掛かりにせず、教材図・名称・操作説明を必ず対応させる。詳細は `docs/visual_design_v1.md` を正本とする。

## 体験目標

プレイヤーは、自分の操作と考えで工房の装置を動かす。式を連続して解くのではなく、分類、点灯、数枠、数直線、位取り、時計、計測、図形組立、グラフ、関係図など、その単元に合う操作を中心にする。

文章やキャラクターは、修理目的と考え方を短く伝えるために使う。ダジャレや学習と無関係な物語を動機の中心にしない。

工房中央の中枢装置「ルミナ」が停止している。プレイヤーは新しい工房の親方となり、トトとモクモと一緒に、考えることで各区画のラインへ光・色・動きを戻す。小学1年生では、6ラインの復旧コアを作ってルミナへ「はじまりの光」を戻すところまでを一つの物語として完結させる。

## 学習ライン

小学1年生は次の6本。各ラインは11ステージ。

1. かず 基礎ライン
2. たしざん 組み立てライン
3. ひきざん 整理ライン
4. くらべる 計測ライン
5. かたち 設計ライン
6. しらべる 解決ライン

小学2年生は次の6本。各ラインは11ステージ。

1. 大きなかず 位取りライン
2. たしひき 筆算ライン
3. かけ算 増幅ライン
4. はかる 計測ライン
5. かたち 設計ライン2
6. 調査・問題解決ライン

各ラインの進捗、ひらめき印、修理パーツ、統計、タイムアタック記録は独立して保存する。未習単元を一律にロックせず、各ラインのステージ1は最初から選べる。ライン内だけ前ステージ完了による順次解放を行う。

## 主要画面

1. 初回案内：ルミナ停止、思考が修理エネルギーになること、親方への任命、工房名設定
2. コース選択：小学1年生・2年生から担当区画を選び、区画の復旧状態を見る
3. 区画導入：選択学年の目的と6本の修理ラインを伝える
4. 工房コントロール室：ルミナの現在状態、6コアの進捗、今日のおすすめ
5. ひらめき設計図：選択ラインの固有ミッション、3区画・11ステージ
6. 修理ベンチ：ステージの修理目的、問題、視覚装置、操作、ヒント
7. 修理レポート：得点、印、時間、パーツ、ステージ固有の世界変化、ライン・学年完成、自己評価
8. 修理パーツ図鑑：選択学年の6ライン・全66パーツと復旧コア
9. おうちの方へ：学年・ライン別の進捗、正答率、タイム記録
10. ひらめきタイムアタック：ライン完走後の12問チャレンジ

小学1年生のライン導入は各ライン初回、ステージ導入は各ステージ初回だけ自動表示する。設計図からライン物語を再読できる。ライン完成は11個目の初回クリア時、学年完成は66個目の初回クリア時だけ発生し、その後は任意で再読できる。

## ライン共通構造

- A区画・ステージ1〜4：具体物と基礎操作
- B区画・ステージ5〜8：確認と表現の組み替え
- C区画・ステージ9〜11：応用、日常場面、総合確認
- ステージ5：直前4ステージの確認
- ステージ11：ライン全体の総合確認

通常ステージは8問で構成する。小学1年生は順番を固定し、ラウンド番号で入力装置を切り替えない。

1. 導入：具体物と小さい数
2. 導入：同じ操作でもう一度
3. 展開：数・配置を広げる
4. 展開：自分で操作して確かめる
5. 場面：文章・生活場面へ置き換える
6. ひねり：等しい場合、向き、途中の数などを確かめる
7. 確認：中心となる考えを再確認する
8. 代表問題：そのステージの到達点を確かめる

学習ステージは1ステージ1中心操作とする。同じ能力を測る間は、操作を固定して数値・配置・場面を変える。例外は時計の「読む／合わせる」のような、読む課題と作る課題が相補関係にあるペアだけとする。ステージ5・11は明示された総合確認のため複数課題を扱えるが、回答の確定文法は変えない。

## 操作エンジン

回答操作は次の共通形式を使い、学習課題と一致するものだけを選ぶ。`stage.action` と `stage.questionContract` を出題生成時に参照し、ラウンド乱数だけで操作を決めない。

- まる・棒・グラフの印を必要数だけタップ
- 見た目の特徴で仲間へ分類
- 二つの量を一対一で比較
- 数枠や10枠を埋める
- 数字を順番に並べる
- 数直線を実際に一目盛りずつ進む・戻る
- 同じ数ずつの分け方を盤面で試す
- 数値を増減して表現を作る
- 時計の長針・短針を動かす
- 図形盤や位置盤のマスを選ぶ

問題文の動詞と操作を一致させる。「とる」はタップで取り除く、「ならべる」は順番に選ぶ、「合わせる」は針や数を動かす、という契約を持つ。操作説明は必須で、推測すべき正解を説明文へ書かない。

iPadで安定して操作でき、キーボードでも代替できるよう、ドラッグ必須にはせず「部品をタップして置き場所を選ぶ」方式を基本とする。

## 回答と評価

- 1ステージ8問
- 最初の回答が正解なら1点
- 最初の回答が不正解なら、具体的なヒントを表示して同じ問題へ再挑戦
- 二回目も不正解なら、正解を含む解説を表示して次へ進める。同じ問題の無限ループにしない
- 再挑戦の正解は称賛するが、ステージ得点には加えない
- 5点以上で修理完了
- 5〜6点：印1個 / 7点：印2個 / 8点：印3個
- 不正解を罰、攻撃、嘲笑として表現しない

## ランダム問題生成

問題は学習ライン・ステージの制約内で、プレイ開始時にseed付きで生成する。

- 数値、対象物、配置、左右、文章場面、選択肢順をランダム化
- 全乱数を注入されたseed付きrngから取得し、同じseedで同じ問題を再現する
- 1プレイ8問のsemantic signature・content signature・learning signatureを重複させない
- ステージごとに直近32問分の3署名、計96キーを端末へ保存し、再挑戦時の重複を避ける
- タイムアタックは直近36問分の3署名、計108キーを避ける
- プレイseedを履歴へ保存し、問題の再現調査を可能にする
- 有限教材のため永久に一度も重ならないことは保証せず、直近プレイとの重複回避を保証対象とする

数値問題の誤答は正解±nの無作為生成だけにしない。足し引きの混同、片方の数をそのまま答える、途中結果、1ずれ、位の取り違えなど、学習上想定される考え方から作る。数値選択問題は原則4択を保証し、正解との衝突で選択肢数を減らさない。意図した0の学習以外では「−0」「全部引く」「あと0」を出さない。

## ひらめきタイムアタック

各ラインの11ステージをすべて修理すると、そのラインだけ解放する。

- 毎回ランダムな12ミッション
- スタートボタンで計測開始
- ミスした問題は同じ操作へ再挑戦
- ミス1回につき公式タイムへ3秒加算
- 画面が非表示の時間は計測から除外
- 操作タイム、ミス数、公式タイム、自己ベスト、挑戦回数をライン別に保存
- 通常ステージの進捗や印には影響しない

## データ

- 保存キー：`lumina_state_v1`。既存記録保護のため変更しない
- state version：4
- 旧v1・v2・v3をv4へ移行し、旧記録を小学1年生コースへ保持する
- 初回v4移行前の生データを `lumina_state_v1_pre_v4` へ一度だけバックアップする
- 将来versionの記録は上書きせず、最新版への更新を案内する
- `courses.g1` と `courses.g2` へ進捗、統計、最近問、履歴、タイムアタックを完全分離する
- 物語改訂番号は `settings.storyRevision`、学年・ライン・ステージ・学年完了の既読は各コースの `storySeen` へ名前空間付きで保存する
- 旧v4利用者には改訂された全体導入を一度だけ再提示し、既存進捗を変更しない
- 既存の足し算・引き算ステージIDを維持する
- 内部の `islandId` は旧履歴互換の別名としてのみ維持する
- 保存内容：工房名、進捗、パーツ、自己評価、設定、ライン統計、出題履歴、タイムアタック記録
- 保護者画面からJSON形式で記録を書き出せる
- 全削除は確認画面を経る

## iPad / PWA・アクセシビリティ

- standalone表示、safe area、縦横両向きへ対応
- 主要ボタンは48px以上
- タップ操作と縦スクロールを両立する
- フォーカスを明示し、色だけで状態を伝えない
- 音を消せる。音なしでも必要情報が分かる
- アプリ内設定と `prefers-reduced-motion` の両方で動きを減らせる
- Service Worker更新時にプレイ中の回答を失わない
- 旧キャッシュ削除は `hirameki-kobo-` 接頭辞のキャッシュだけを対象にする
