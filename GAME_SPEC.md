# ゲーム仕様 — ひらめき工房 v13

## 製品概要

- タイトル：ひらめき工房
- 対象：小学1年生
- 学習範囲：算数6ライン・全66ステージ
- 形式：ブラウザで動くPWA
- 主対象端末：iPad。スマートフォンとPCにも対応
- 公開先：<https://hassy0511.github.io/hirameki-kobo/>
- 外部ライブラリ、外部画像、学習中の外部通信は使用しない

このv13仕様は現在稼働中の小学1年生版を表す。小学2年生は `docs/curriculum_math_g2_overview.md` と `grade2-curriculum.js` で設計済みだが、学年別state v4と学年選択を入れるまで実行時には接続しない。

## 体験目標

プレイヤーは、自分の操作と考えで工房の装置を動かす。式を連続して解くのではなく、分類、点灯、数枠、数直線、位取り、時計、計測、図形組立、グラフ、関係図など、その単元に合う操作を中心にする。

文章やキャラクターは、修理目的と考え方を短く伝えるために使う。ダジャレや学習と無関係な物語を動機の中心にしない。

## 学習ライン

次の6本をすべて実装する。各ラインは11ステージ。

1. かず 基礎ライン
2. たしざん 組み立てライン
3. ひきざん 整理ライン
4. くらべる 計測ライン
5. かたち 設計ライン
6. しらべる 解決ライン

各ラインの進捗、ひらめき印、修理パーツ、統計、タイムアタック記録は独立して保存する。未習単元を一律にロックせず、各ラインのステージ1は最初から選べる。ライン内だけ前ステージ完了による順次解放を行う。

## 主要画面

1. 初回案内：工房の説明と工房名設定
2. 工房コントロール室：今日のおすすめと6ラインの進捗
3. ひらめき設計図：選択ラインの3区画・11ステージ
4. 修理ベンチ：問題、視覚装置、操作、ヒント
5. 修理レポート：得点、印、時間、パーツ、自己評価
6. 修理パーツ図鑑：6ライン・全66パーツ
7. おうちの方へ：全体・ライン別の進捗、正答率、タイム記録
8. ひらめきタイムアタック：ライン完走後の12問チャレンジ

## ライン共通構造

- A区画・ステージ1〜4：具体物と基礎操作
- B区画・ステージ5〜8：確認と表現の組み替え
- C区画・ステージ9〜11：応用、日常場面、総合確認
- ステージ5：直前4ステージの確認
- ステージ11：ライン全体の総合確認

通常ステージは8問で構成する。操作問題を中心にし、場面・文章問題と確認問題を混ぜる。足し算・引き算でも式だけの問題が連続しないようにする。

## 操作エンジン

回答操作は次の共通形式を使い、視覚装置を単元ごとに切り替える。

- 部品を必要数だけタップ
- 部品をトレイへ分類
- 二つの量を一対一で比較
- 数枠や10枠を埋める
- 数字車両を順番に並べる
- 数直線・回路・ルートを選ぶ
- ギアの数値を増減する
- 時計の長針・短針を動かす
- 図形盤や位置盤のマスを選ぶ

iPadで安定して操作でき、キーボードでも代替できるよう、ドラッグ必須にはせず「部品をタップして置き場所を選ぶ」方式を基本とする。

## 回答と評価

- 1ステージ8問
- 最初の回答が正解なら1点
- 最初の回答が不正解なら、具体的なヒントを表示して同じ問題へ再挑戦
- 再挑戦の正解は称賛するが、ステージ得点には加えない
- 5点以上で修理完了
- 5〜6点：印1個 / 7点：印2個 / 8点：印3個
- 不正解を罰、攻撃、嘲笑として表現しない

## ランダム問題生成

問題は学習ライン・ステージの制約内で、プレイ開始時にseed付きで生成する。

- 数値、対象物、配置、左右、文章場面、選択肢順をランダム化
- 1プレイ8問のsemantic signatureを重複させない
- ステージごとに直近32問のsignatureを端末へ保存し、再挑戦時の重複を避ける
- タイムアタックは直近36問を避ける
- プレイseedを履歴へ保存し、問題の再現調査を可能にする
- 有限教材のため永久に一度も重ならないことは保証せず、直近プレイとの重複回避を保証対象とする

## ひらめきタイムアタック

各ラインの11ステージをすべて修理すると、そのラインだけ解放する。

- 毎回ランダムな12ミッション
- スタートボタンで計測開始
- ミスした問題は同じ操作へ再挑戦
- ミス1回につき公式タイムへ3秒加算
- 画面が非表示の時間は計測から除外
- 操作タイム、ミス数、公式タイム、自己ベスト、挑戦回数をライン別に保存
- 通常ステージの進捗や印には影響しない

## データ

- 保存キー：`lumina_state_v1`。既存記録保護のため変更しない
- state version：3
- 旧v1・v2をv3へ移行する
- 既存の足し算・引き算ステージIDを維持する
- 内部の `islandId` は旧履歴互換の別名としてのみ維持する
- 保存内容：工房名、進捗、パーツ、自己評価、設定、ライン統計、出題履歴、タイムアタック記録
- 保護者画面からJSON形式で記録を書き出せる
- 全削除は確認画面を経る

## 実装構成

- `index.html`：PWAの入口
- `styles.css`：画面、装置、操作、レスポンシブ表示
- `game-core.js`：66ステージ定義、問題生成、保存移行、タイムアタック問題
- `app.js`：画面、操作、評価、保存、タイマー、PWA更新
- `tests/smoke-v13.mjs`：全生成、全完走、移行、タイムアタック、PWA検査

## iPad / PWA・アクセシビリティ

- standalone表示、safe area、縦横両向きへ対応
- 主要ボタンは48px以上
- タップ操作と縦スクロールを両立する
- フォーカスを明示し、色だけで状態を伝えない
- 音を消せる。音なしでも必要情報が分かる
- アプリ内設定と `prefers-reduced-motion` の両方で動きを減らせる
- Service Worker更新時にプレイ中の回答を失わない
- 旧キャッシュ削除は `hirameki-kobo-` 接頭辞のキャッシュだけを対象にする
