# 小学3年生を安全に追加する実装計画

## 結論

小学3年生は `grade3-curriculum.js` に7ライン・77ステージの学習メタデータまで準備済みである。公開アプリへはまだ接続しない。

小学1・2年生については `COURSES.g1/g2`、state v4、学年選択、二年生問題ビルダー、全完走検査まで完成している。この共通APIを7ラインへ一般化し、三年生を追加する。

## 正本

- 小学1年生：現行 `game-core.js` の6ライン・66ステージ
- 小学2年生：`grade2-curriculum.js` の6ライン・66ステージ
- 小学3年生：`grade3-curriculum.js` の7ライン・77ステージ
- 三年生の調査・設計根拠：`docs/curriculum_math_g3_overview.md`

## 学年レジストリ

```js
const COURSES = {
  g1: GRADE1_CURRICULUM,
  g2: HiramekiGrade2Curriculum,
  g3: HiramekiGrade3Curriculum
};
```

正式APIは配列indexではなく、`gradeId + lineId + stageId` で解決する。

```js
courseFor(courseId)
lineFor(courseId, lineId)
makeStageQuestions(courseId, lineId, stageId, options)
makeTimeAttackQuestions(courseId, lineId, options)
isUnlocked(state, courseId, stageIndex, lineId)
```

三年生だけ7ライン・77ステージなので、「全学年6ライン」「各学年66ステージ」を前提にしない。ライン内は11ステージだが、画面の総数、進捗率、印の最大数、タイムアタック解放条件はカリキュラム定義から計算する。

## 保存形式

state v4以降は、`courses` を動的に作る。

```js
{
  version: 4,
  activeCourseId: 'g1',
  courses: {
    g1: createCourseState('g1'),
    g2: createCourseState('g2'),
    g3: createCourseState('g3')
  }
}
```

各学年状態には次を持つ。

- `lastLine`
- `progress`
- `parts`
- `moods`
- `stats`
- `lineStats`
- `lineIntros`
- `timeAttack`
- `recentQuestions`
- `recentRush`
- `history`

新しい学年が定義に増えたときは、既存stateへ空の学年状態を追加する。既存のG1・G2データを作り直したり、未知の学年をG1として扱ったりしない。

## 三年生で必要になる問題操作

二年生までの共通操作に、次を追加・拡張する。

- 3・4位数筆算の連続繰り上がり・繰り下がり
- アレイ分割、部分積レーン、2位数乗数の筆算修理
- 等分配布と指定数パッキングを分けた除法操作
- 余りポケットと、場面に合う余りの扱い
- 0.1チップ、小数点固定の位取り、分数片、小数・分数の数直線重ね合わせ
- 秒タイマー、時計と時間数直線の連動、巻尺、地図ルート、はかり
- コンパス、円の半径アーム、球の切断、角の重ね合わせ
- 棒グラフの目盛り切替、簡単な表の組合せ、根拠と発見文の対応
- 四則演算レバー、□を置く関係図、逆向きの計算回路

iPadではHTML5 drag-and-dropへ依存せず、「部品をタップして選ぶ→置き場所をタップする」を共通方式にする。操作途中の状態はDOMではなく問題オブジェクトへ保持する。

## 問題ビルダー

```js
const BUILDERS = {
  g1: G1_BUILDERS,
  g2: G2_BUILDERS,
  g3: {
    number: {},
    multiplication: {},
    division: {},
    decimalFraction: {},
    measure: {},
    shape: {},
    solve: {}
  }
};
```

各ステージは通常8問。操作問題5問以上、短い文章・生活場面1〜2問、表現変換2問以上、式だけの計算は最大1問とする。ステージ5と11は複数技能を混ぜた確認テストにする。

生成時は数値だけでなく、対象物、配置、向き、文章、正解位置をseed付きで変える。`gradeId/lineId/stageId/semanticSignature` を組にし、同一8問内と直近32問の重複を避ける。

## 境界をコードで守る

問題ビルダーには次の制約をテスト可能な形で持たせる。

- 整数は一億まで
- 乗法は最大3位数×2位数
- 除法の商2位数は、何十を割るか各位が割り切れる簡単な場合だけ。一般的な除法筆算を生成しない
- 小数は小数第一位の加減だけ
- 分数加減は真分数どうし、同分母、和が1までとその逆
- 角は重ね合わせによる比較だけ。度・分度器を生成しない
- データは棒グラフと簡単な組合せ表まで。折れ線グラフを生成しない
- □は未知量一つだけ

## 実装順

1. 完成済みのG1回帰、`COURSES.g1/g2`、state v4、G2全完走を固定する。
2. コースレジストリとUIを、G3の7ライン・77ステージへ一般化する。
3. 三年生の加減・かけ算・わり算を実装する。
4. 小数・分数、計測、図形、調査・問題解決を実装する。
5. 三年生7ラインの12問制タイムアタックを接続する。
6. 学年選択、ホーム、設計図、図鑑、保護者画面を77ステージへ動的対応させる。
7. PWAキャッシュを更新し、HTTPS上で更新切替・オフライン・iPad操作を確認する。
