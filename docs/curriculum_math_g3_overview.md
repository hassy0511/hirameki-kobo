# 小学3年生 算数カリキュラム全体構成

## 結論

小学3年生は、7本の学習ライン・各11ステージ、合計77ステージで構成する。

一年生・二年生と同じ6本へ詰めると、三年生で新しく本格導入される除法と、小数・分数のどちらかが別領域へ埋もれる。そこで、A「数と計算」を次の4ラインに分け、B・C・Dをそれぞれ独立させる。

| ライン | 主な領域 | ステージ数 |
|---|---|---:|
| 大きな数・たしひき 制御ライン | A(1) 数、A(2) 加減、A(8) そろばん | 11 |
| かけ算 増幅ライン3 | A(3) 乗法 | 11 |
| わり算 分配ライン | A(4) 除法 | 11 |
| 小数・分数 精密ライン | A(5) 小数、A(6) 分数 | 11 |
| はかる 計測ライン3 | C(1) 長さ・重さ、C(2) 時刻・時間 | 11 |
| かたち 設計ライン3 | B(1) 図形 | 11 |
| 調査・問題解決ライン3 | D(1) データ、A(7) □を用いた式 | 11 |

ステージ1〜4は基礎作業、5は前半確認、6〜8は組み替え、9〜10は応用、11はライン総合確認とする。これは画面上の共通構造であり、教科書の1単元を機械的に1ステージへ対応させたものではない。

三社で年度前半の並びは大きく異なるため、一社の目次をそのまま正本にはしない。共通技能をライン内で段階化し、出版社差は「年間おすすめ順」と学校進度の表示で吸収する。別ライン間は一律にロックしない。

## 三年生で越えない境界

| 領域 | 三年生で扱う | 四年生以降へ送る |
|---|---|---|
| 整数 | 万、一億まで、10・100・1000倍、10分の1、相対的な大きさ | 何億・何兆という体系的な扱い |
| 加減 | 3・4位数の筆算、簡単な2位数暗算、感覚的な見積り | 四捨五入、概数の形式的な計算 |
| 乗法 | 2・3位数×1・2位数、筆算、性質、暗算、見積り | 3位数×3位数、小数乗法 |
| 除法 | 除数・商1位数、余り、80÷4・69÷3など簡単な商2位数 | 一般的な除法筆算、2位数除数 |
| 小数 | 小数第一位の意味・大小・加減 | 小数第二位以降、小数の乗除 |
| 分数 | 単位分数、同分母の大小、真分数どうしで和1までの加法と逆の減法 | 仮分数、帯分数、和が1を超える計算、異分母計算 |
| 角 | 形としての角、重ねた大小・相等、三角形の角の性質 | 度、分度器による角度測定 |
| 図形 | 円・球、二等辺・正三角形、コンパス | 平行・垂直、平行四辺形等、直方体・立方体、面積 |
| 測定 | 秒、km、g・kg・t、適切な単位・計器、正味と容器、既習単位を通した接頭語k・mへの接触 | 複雑な換算、未知の単位を使う換算ドリル |
| データ | 棒グラフ、一次元表を組み合わせた簡単な表 | 本格的な二次元表、折れ線グラフ |
| □の式 | 未知量一つを□で表し、図と四則の相互関係で求める | 二つの変量、文字式、関数的な扱い |

除法の「商が2位数」は特に広げすぎない。`80÷4`のように何十を割る場合と、`69÷3`のように十の位部分と一の位部分がそれぞれ割り切れる場合に限定し、縦書きの一般除法筆算は入れない。

## 1ステージ8問の構成

通常ステージは次を最低基準とする。

- 操作を伴う問題：5問以上
- 短い文章・生活場面：1〜2問
- 具体物・図・数直線・表・式の表現変換：2問以上
- 式だけの計算：最大1問
- ステージ5：直前4技能を混ぜた前半確認
- ステージ11：ライン全体を混ぜた総合確認

数値、対象物、配置、向き、文章場面、正解位置をseed付きで変える。semantic signatureを作り、同一8問内の重複を禁止し、同じステージの直近32問も可能な限り避ける。

筆算でも、答えを数字で入力するだけにはしない。位をそろえる、部分積を置く、繰り上がり札を動かす、故障した途中式を直す、見積り範囲を選ぶ、場面と式を対応させる操作を混ぜる。

## タイムアタック

各ライン11ステージを完了すると、その学年・そのライン専用の12問制タイムアタックを解放する。

- ステージ5・11の確認問題は直接出題元にせず、1〜4・6〜10の短縮操作版から12枠を組む
- 毎回seedを変え、数値、対象、配置、正解位置をランダム化する
- ラインごとに技能枠を固定し、毎回の難易度と操作量をそろえる
- 長文読解速度が記録を支配しないよう、「短文＋図＋操作」にする
- ミス1回につき3秒加算する
- 実測タイム、加算後の公式タイム、ミス数、自己ベストを学年・ライン別に保存する

| ライン | 12枠のステージ配分 |
|---|---|
| 大きな数・たしひき | `1,2,3,4,1,2,6,7,8,9,10,7` |
| かけ算 | `1,2,3,4,6,6,7,7,8,9,9,10` |
| わり算 | `1,2,3,4,6,6,7,7,8,8,9,10` |

## 実装ルール

- 学年レジストリを `CURRICULA.g1` / `CURRICULA.g2` / `CURRICULA.g3` とする
- 既存G1・G2のstage IDを変更しない
- 三年生のstage IDはすべて `g3_`、技能IDはすべて `g3.` で始める
- 保存データは学年階層で完全分離し、問題、セッション、履歴、最近問、タイムアタックへ必ず `gradeId` を持たせる
- 学年ごとのライン数・ステージ数をハードコードせず、三年生の7ライン・77ステージも動的に描画する
- 不明な学年・ライン・ステージを一年生へ黙ってフォールバックしない
- 問題ビルダーを接続するまでは `grade3-curriculum.js` を公開アプリの実行資産へ含めない

実装用メタデータは `grade3-curriculum.js`、単体定義検査は `tests/grade3-curriculum-smoke.mjs` を正とする。
