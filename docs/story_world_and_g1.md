# ひらめき工房 — 全体ストーリー設計と小学1年生実装

## 1. この文書の位置付け

この文書は、作品全体の物語構造と、小学1年生コースへ実装する物語内容の正本である。学習順序と出題範囲は各カリキュラム文書、キャラクターと体験原則は `WORKSHOP_BIBLE.md` を優先する。

物語は飾りではなく、「何のために問題へ取り組むのか」「正解すると何が変わるのか」「一つのラインを終えると何を達成したのか」を子どもへ伝える進行装置として扱う。ただし、物語の知識がなければ算数問題を解けない構造にはしない。

## 2. 全体の物語

ひらめき工房の中央には、各学年区画の学習ラインへ力を送る中枢装置「ルミナ」がある。ルミナの回路が外れ、工房の光、ベルト、計器、設計機、案内盤が止まった。

外れた回路はボタンを押すだけでは戻らない。分類する、数える、合わせる、取り出す、比べる、組み立てる、情報をつなぐといった思考が修理エネルギーになる。

プレイヤーは新しい「工房の親方」として工房名を決め、トトとモクモと一緒に装置を直す。敵や敗北は置かず、自分の理解が工房の光・色・動きとして返ってくることを中心にする。

## 3. 登場役

- プレイヤー：ひらめき工房の親方。装置を操作し、考えて修理方法を決める
- トト：見方を案内する工房アシスタント。答えを先に言わず、次に見る場所を短く伝える
- モクモ：装置の整備担当。故障箇所と修理目的を伝える。不正解を笑ったり責めたりしない
- ルミナ：工房中央の中枢装置。人格を持つ案内役ではなく、学びによって状態が変わる修理対象

## 4. 学年と物語の関係

学年は物語上の強制順ではなく、それぞれ独立して選べる修理区画とする。

- 小学1年生「はじまり区画」：六つの基礎コアを作り、ルミナへ「はじまりの光」を戻す
- 小学2年生「ひろがり区画」：大きな数、筆算、九九をつなぎ、大型装置を動かす
- 将来の学年：その学年だけでも始まりと達成が分かる独立区画として追加する

小学2年生から始めても物語上の矛盾を起こさない。小学1年生完了後も、次学年への強制移動は行わない。

## 5. 画面をまたぐ物語ビート

| 場所 | 子どもへ伝えること | 表示条件 |
|---|---|---|
| 全体導入 | ルミナ停止、思考が修理エネルギー、プレイヤーは親方 | 新規利用時。物語改訂時は既存利用者にも一度再提示 |
| コース選択 | どの学年区画を直すか、各区画の目的と復旧状態 | いつでも再訪可能 |
| 学年導入 | その区画で戻す機能、ライン数、自由な開始順 | 学年ごとに初回1回 |
| 工房ホーム | ルミナの現在状態、完成コア数、次の修理 | 常時。進捗から動的生成 |
| ライン導入 | 固有の故障、戻す力、11パーツの目的 | ラインごとに初回1回。設計図から再読可能 |
| ステージ導入 | 今回直す装置、親方が行う操作 | ステージごとに初回1回 |
| 修理ベンチ | 正解で対象回路へエネルギーが届く | 各問題中 |
| 修理レポート | 獲得パーツと、装置・工房に起きた固有の変化 | 初回クリア時。再クリアは短い調整完了表示 |
| ライン完了 | 11パーツが一つのコアになり、ライン全体が復旧 | 11個目の初回クリア時のみ |
| タイムアタック | 復旧済みラインの任意の高速点検 | ライン完了後 |
| 学年完了 | 六つのコアがルミナへ接続され、区画が復旧 | 66個目の初回クリア時。後から再読可能 |

## 6. 小学1年生の六つの物語ライン

| ライン | 止まった装置 | 戻す力 | 完了時の変化 |
|---|---|---|---|
| かず 基礎 | 材料管理装置 | 見つけて数える力 | 材料の名前・数・順番がそろい、全ラインへ正しい部品を届ける |
| たしざん 組み立て | 合流エネルギー装置 | 合わせて大きくする力 | 二本の光が一つになり、ルミナへ強いエネルギーを送る |
| ひきざん 整理 | 整理・搬出装置 | 取り出して整える力 | 倉庫の詰まりが消え、必要な部品をすぐ運び出す |
| くらべる 計測 | 計測・時刻装置 | 比べてそろえる力 | 橋、タンク、時計が同じ合図で動く |
| かたち 設計 | 形・設計装置 | 形を見つけて作る力 | 足場、壁、枠、標識が正しい形と位置へ組み上がる |
| しらべる 解決 | 情報・解決装置 | 手がかりをつなぐ力 | カード、グラフ、図、式がつながり、全ラインへ指示を送る |

各ラインの全11ステージには、取得パーツと対応する固有の工房変化を一つずつ割り当てる。実行データの正本は `story-core.js` とし、学習ステージIDとは配列位置と検証テストで対応を保証する。

## 7. 一年生の結末

六ライン・66ステージが初めてそろったとき、六つのコアがルミナへ接続される。

- トトが六つの力の接続を確認する
- モクモが「はじまり区画、完全復旧」を報告する
- 「はじまりの光」の復旧エンブレムを表示する
- 「この区画を遊びこむ」「ほかの区画を見る」を選べる状態へ戻す

次学年を強制的に開始しない。完了後のステージ再挑戦、ひらめき印集め、ライン別タイムアタックを残す。

## 8. コピーと演出の基準

- 一つのイベントは最大2〜3場面、1場面の本文は原則70字以内
- 算数の操作説明を物語で曖昧にしない
- 正解時は数学的な説明を先にし、装置の反応は短く添える
- 不正解時は具体的な見方を示し、罰、損壊、嘲笑を使わない
- 毎問長い会話を挟まず、導入・節目・結果へ物語を集中させる
- 学習内容と無関係な固有名詞や設定を問題文へ混ぜない

## 9. 保存と既存利用者

- 保存データのstate versionは4のまま維持する
- 物語改訂番号は既存の `settings.storyRevision` に保存し、古いアプリでも未知設定として保持できるようにする
- 学年・ライン・ステージ・学年完了の既読は、既存の `courses[courseId].storySeen` へ名前空間付きキーで保存する
- 旧 `introSeen`、`lineIntros`、ステージID直下の `storySeen` は互換のため維持する
- 物語更新で進捗、印、パーツ、履歴、最近問、タイムアタック記録を変更しない

## 10. 実装完了条件

- 初見で「ルミナを直す」「自分は工房の親方」「考えると装置が動く」が分かる
- 小学1年生6ラインに別々の故障・目的・復旧機能がある
- 小学1年生全66ステージに別々の世界変化がある
- ライン完成と学年完成が、再プレイのたびに繰り返されない
- 既存利用者にも新しい全体導入が一度表示され、進捗は保持される
- 物語の既読状態がランダム問題のseed、内容、順序へ影響しない
- 小学2年生の進捗・出題・タイムアタックへ影響しない
