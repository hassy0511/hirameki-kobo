## 二年生で越えない境界

| 領域 | 二年生で扱う | 今回は扱わない |
|---|---|---|
| 整数 | 9999までの4位数と10000、位取り、数直線、＞・＜、10・100単位の見方 | 10000を超える一般的な数 |
| 加減 | 2位数どうしの筆算、和が3位数になる加法、その逆の減法、簡単な3位数と1・2位数の加減 | 一般的な3位数どうしの筆算 |
| 乗法 | 乗法の意味、九九、倍、交換法則など、10〜12程度を含む簡単な乗法 | 一般的な2位数×1位数の筆算 |
| 分数 | 具体物を等分して作る1/2、1/3、1/4など | 分数の大小比較、加減計算 |
| 測定 | mm・cm・m、mL・dL・L、時・分・日、午前・午後 | 秒、重さ、面積の公式 |
| 図形 | 三角形・四角形、直角、長方形、正方形、直角三角形、箱の素地 | 形式的な立方体・直方体の定義、本格的な展開図 |
| データ | 一つの観点による分類、簡単な一次元表、○などを並べたグラフ | 二次元表、正式な棒グラフ |

特に「簡単な3位数の加減」は、`628＋7`、`234＋57`、`753－6`、`683－51`のように百の位で複雑な繰り上がり・繰り下がりを起こさない範囲を基本とする。九九を超える乗法も、`4×10`、`4×11`、`4×12`やその交換に当たる程度とする。

## 1ステージ8問の構成

通常ステージは次を最低基準とする。

- 操作を伴う問題：5問以上
- 短い文章・生活場面：1〜2問
- 図・表・式・具体物の表現変換：2問以上
- 式だけの計算：最大1問
- ステージ5：直前4技能を混ぜた前半確認
- ステージ11：ライン全体を混ぜた総合確認

問題は数値だけでなく、対象物、配置、向き、文章場面、正解位置をseed付きで変える。semantic signatureを作り、同一8問内の重複を禁止し、同じステージの直近32問も可能な限り避ける。

## 文章題の扱い

文章題は「最後に計算式を一問置く」扱いにしない。

1. 場面の動きを再生する
2. 必要な数や情報を選ぶ
3. まとまり図、位取りブロック、テープ図、表などへ置く
4. 演算や式を選ぶ
5. 答えが場面に合うか確かめる

調査・問題解決ラインでは文章題そのものを技能として扱う。他の5ラインでも毎ステージに短い生活場面を入れ、単位や図形、九九が実生活で何を表すかを確認する。

## タイムアタック

各ライン11ステージを完了すると、その学年・そのライン専用の12問制タイムアタックを解放する。

- 毎回seedを変え、数値、配置、対象物、正解位置をランダム化
- ラインごとに12個の技能枠を固定し、毎回の難易度と操作量をそろえる
- 長文や組立工程の長い問題は短縮版を使う
- ミス1回につき3秒加算
- 実測タイム、加算後の公式タイム、ミス数、自己ベストを学年・ライン別に保存
- 小学1年生の自己ベストとは完全に分離

## 年間おすすめ順

| 時期 | おすすめ | 主な対応単元 |
|---|---|---|
| 1学期前半 | 調査1〜5 → 筆算1〜5 | 表・グラフ、2位数の加減 |
| 1学期後半 | 計測1〜9 → 大きな数1〜5 | cm・mm、1000まで、かさ、時刻・時間 |
| 2学期前半 | 筆算6〜10 → 図形1〜9 | 発展筆算、計算の工夫、三角形・四角形 |
| 2学期後半 | かけ算1〜8 | 乗法の意味、1〜9の九九 |
| 3学期 | かけ算9〜11、計測10〜11、大きな数6〜11、解決6〜11、図形10〜11、筆算11 | 九九の性質、m、10000、逆思考、箱、分数、総復習 |

学校や採択教科書によって順序が違うため、別ライン同士を一律にはロックしない。画面上の「おすすめ」を季節や学校進度に合わせて変え、ライン内だけ段階解放する。

## 小学1年生からの接続

| 二年生ライン | 主な一年生前提 |
|---|---|
| 大きな数 | 100程度までの位取り、数直線、2・5・10ずつ数える、同じ数ずつ分ける |
| たしひき筆算 | 20までの加減、10を作る・10から引く、100程度の数構成 |
| かけ算 | 同じ数ずつのまとまり・等分、加法、飛び数え |
| 計測 | 直接・間接・任意単位による比較、何時何分 |
| 図形 | 立体の分類、面の形、色板・棒・点による構成、位置 |
| 調査・問題解決 | 分類、絵グラフ、加減の演算決定、全体と部分の図 |

二年生ステージ開始時に一年生の前提を長い復習単元として繰り返さず、最初の操作で短く確認する。つまずきがあれば対応する一年生の `canonicalSkillId` へ戻れるようにする。

## 実装ルール

- 学年レジストリを `CURRICULA.g1` / `CURRICULA.g2` とする
- 既存小学1年生の66個のstage IDは変更しない
- 二年生のstage IDはすべて `g2_`、技能IDはすべて `g2.` で始める
- 保存データはstate v4で `grades.g1` / `grades.g2` に完全分離する
- 問題、セッション、履歴、タイムアタック、最近問へ必ず `gradeId` を持たせる
- 不明な学年・ライン・ステージを一年生へ黙ってフォールバックしない
- 学年選択、ホーム、設計図、図鑑、保護者画面の総数はデータから動的に計算する
- 小学1年生へ戻ったとき、進捗、印、パーツ、履歴、タイムアタックが追加前と一致することを回帰テストする

具体的な移行手順は `docs/implementation_math_g2.md` を正とする。
